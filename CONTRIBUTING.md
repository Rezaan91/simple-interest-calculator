# Contributing

All contributions, bug reports, bug fixes, documentation improvements, enhancements, and ideas are welcome.

## How to contribute
1. **Fork** the repository to your GitHub account.
2. Create a feature branch from `main` (e.g., `bug-fix-typo`).
3. Make small, focused changes and include tests/examples where relevant.
4. Ensure the script still runs and outputs correct results.
5. **Open a Pull Request** describing what changed and why.

## Reporting issues
- Use a clear, descriptive title and steps to reproduce.
- Include OS/shell, script version, and any logs.

## Commit style
- Use descriptive commit messages (imperative mood).
- Reference issues/PRs where applicable.

## Code of Conduct
By participating, you agree to abide by our [Code of Conduct](./CODE_OF_CONDUCT.md).
